API documentation

routes:
/ : index.html
/login : login.html
/login/login : login form endpoint POST
/login/signup : signup form endpoint POST
/chat : chat.html
/chat/retrieve : retreive existing chats using JWT, GET
/chat/message : send user message and save current chat message pair POST
/chat/message/topic : append topic to a chat if it has more than two messages OBS NOT USED ANYMORE
/chat/delete : delete chosen chat with POST
/characters : retrieve existing characters
/characters/create : create character
/characters/delete : delete character

NOTE! admin account can see all characters and all messages, and delete them. user accounts can only see accounts that belong to themselves. Example character is included and built-in. if topic marker is unable to generate, the sha256 conversation_id is used instead
